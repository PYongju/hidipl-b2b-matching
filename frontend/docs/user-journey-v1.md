# 견적 검토 코파일럿 유저저니 v1

작성일: 2026-05-28  
작성 목적: 견적 검토 코파일럿의 전체 사용자 흐름을 팀이 같은 기준으로 이해할 수 있도록 정리한다.  
문서 상태: 확정 스펙이 아니라, 화면 설계와 프로토타입 제작을 위한 유저저니 초안이다.

## 1. 제품 한 줄 정의

견적 검토 코파일럿은 사용자가 고객 요청사항과 공급사 견적서를 업로드하면, AI가 동일 기준으로 견적을 비교하고 추천 근거를 정리해 담당자의 최종 선정과 고객 보고서 생성을 돕는 PC 우선 반응형 웹 서비스이다.

## 2. 주요 사용자

### 2.1 구매/견적 검토 담당자

가장 핵심 사용자이다.

주요 목적:

- 이전 프로젝트 확인
- 새 견적 검토 건 생성
- 고객 요청사항 입력
- 공급사 견적서 업로드
- AI 비교 결과 검토
- 비교표 수정
- AI 재분석
- 최종 공급사 선정
- 고객 보고서 생성

### 2.2 팀 리더/검토 승인자

담당자가 정리한 결과를 확인하고 의사결정하거나 외부 공유 전 검토한다.

주요 목적:

- 프로젝트별 진행 상태 확인
- AI 추천 근거 확인
- 최종 선정 이유 확인
- 고객 보고서 확인

## 3. 전체 유저저니

권장 흐름은 아래와 같다.

```text
로그인
→ 프로젝트 히스토리/목록
→ 새 프로젝트 생성
→ 고객 요청사항 입력
→ 견적서 업로드
→ AI 비교 실행
→ AI 분석 중
→ 비교 대시보드
→ 비교표 수정/메모 작성
→ AI 재분석
→ 최종 선정 확정
→ 고객 보고서 내보내기
```

화면 단위로는 아래 6개 화면으로 정리한다.

1. 로그인 페이지
2. 프로젝트 히스토리/목록 페이지
3. 새 프로젝트 생성 페이지 또는 Wizard
4. AI 분석 중 화면
5. 비교 대시보드
6. 보고서 미리보기/내보내기 화면

## 4. 화면별 유저저니 및 스펙 초안

## 4.1 로그인 페이지

### 목적

사용자가 개인 또는 팀 계정으로 서비스에 진입하는 첫 화면이다.

### 주요 요소

- 서비스명: 견적 검토 코파일럿
- 한 줄 설명
- 이메일/비밀번호 입력
- 로그인 버튼
- 필요 시 Google/Microsoft 계정 로그인
- 비밀번호 찾기

### UX 방향

초기 버전에서는 복잡한 랜딩 페이지보다 로그인 중심의 간결한 화면이 적합하다.

권장 구성:

```text
좌측: 제품 설명 또는 주요 가치
우측: 로그인 폼
```

반응형:

- PC: 좌우 2단 구성
- 태블릿: 중앙 카드
- 모바일: 로그인 폼 단일 구성

## 4.2 프로젝트 히스토리/목록 페이지

### 목적

로그인 후 사용자가 이전 프로젝트를 찾고, 새 프로젝트를 생성하는 허브 화면이다.

이 화면은 단순 히스토리 로그가 아니라 프로젝트 단위로 업무 맥락을 복원하는 화면이다.

### 필요한 이유

- 월 4~10건의 견적 검토가 발생하면 6개월 후 50건 이상 쌓일 수 있다.
- 전체 활동 로그 방식은 어떤 기록이 어떤 프로젝트에 속하는지 찾기 어렵다.
- 구매/견적 검토 업무는 한 달 전 상담 맥락을 다시 봐야 하는 경우가 많다.
- 결재 라인 또는 팀 공유 시 프로젝트 단위로 묶여 있어야 이해가 쉽다.

### 주요 요소

- 상단 헤더
  - 서비스명
  - 사용자 정보
  - 새 프로젝트 생성 버튼

- 상태 요약 카드
  - 진행 중 프로젝트 수
  - AI 분석 대기 프로젝트 수
  - 최종 선정 완료 프로젝트 수
  - 이번 달 생성 건수

- 프로젝트 목록
  - 프로젝트명
  - 고객사/회사명
  - 설치 위치
  - 현재 단계
  - 비교 마감일
  - 공급사 수
  - 최근 수정일
  - 담당자
  - 최종 추천 공급사

- 필터/검색
  - 전체
  - 작성 중
  - 견적 업로드 중
  - AI 분석 중
  - 비교 검토 중
  - 최종 선정 완료
  - 보고서 생성 완료

### 프로젝트 카드 예시

```text
PV-2025-0421
삼성전자 반도체 사이니지 구축

상태: 비교 검토 중
비교 마감일: 2025-04-28
공급사: 3개
최종 추천: A Display
최근 수정: 2026-05-28 14:20

[대시보드 열기]
```

### 새 프로젝트 생성 진입

프로젝트 목록 페이지의 가장 중요한 CTA는 `새 프로젝트 생성`이다.

버튼 위치:

- PC: 우측 상단 고정
- 태블릿/모바일: 상단 또는 하단 플로팅 버튼

## 4.3 새 프로젝트 생성 페이지/Wizard

### 목적

고객 요청사항과 비교할 견적서를 입력받아 AI 비교를 실행하는 단계이다.

초기에는 Drawer로 구현할 수 있으나, 입력 정보와 업로드 기능이 많아지면 별도 페이지 또는 넓은 Wizard 화면이 더 적합하다.

### 추천 방식

최종 제품 흐름에서는 별도 Wizard 페이지를 추천한다.

이유:

- 고객 요청사항 입력 항목이 많다.
- 견적서 업로드가 공급사별로 반복된다.
- 비교 기준 설정이 필요하다.
- 모바일/태블릿 대응이 Drawer보다 쉽다.
- 나중에 임시 저장/이어하기 기능을 붙이기 좋다.

### Step 1. 프로젝트 기본 정보

필수 입력:

- 회사명
- 설치 위치
- 프로젝트 일정
- 비교 마감일
- 현재 단계

날짜 입력 방식:

- 프로젝트 일정과 비교 마감일은 직접 텍스트 입력보다 Date Picker로 선택하게 한다.
- 날짜 선택 시 D-day 계산에 사용한다.
- 날짜는 YYYY-MM-DD 형식으로 저장한다.

현재 단계 예시:

- 정보 탐색
- 비교 검토 중
- 발주 직전
- 계약 진행 중

### Step 2. 고객 요청사항 입력

필수 입력:

- 활용 용도
- 디스플레이 요구사항
- 희망 예산
- 희망 납기
- 중요 우선순위

우선순위 예시:

- 최저가 우선
- 납기 우선
- 신뢰도 우선
- 유지보수 우선
- 스펙 우선
- 균형 추천

### Step 3. 견적서 업로드

필수 기능:

- 공급사별 견적서 업로드
- 여러 파일 업로드
- 파일별 공급사명 입력 또는 자동 인식
- 업로드 완료 상태 표시
- 파일 교체
- 파일 삭제

업로드 상태:

- 업로드 전
- 업로드 완료
- OCR/추출 중
- 추출 완료
- 확인 필요

### Step 4. 비교 기준 확인

AI 분석 전에 비교 정책을 확인한다.

기본 정책:

- 동일 카테고리끼리만 비교
- 비요청 옵션은 별도 코멘트 처리
- 단가 결측은 재요청, 추정 금지
- 납기 결측은 낮은 중요도로 처리
- Size/Spec 차이는 비교표에 명기

### Step 5. AI 비교 실행

사용자가 `AI 비교 시작` 버튼을 누르면 분석 중 화면으로 이동한다.

## 4.4 AI 분석 중 화면

### 목적

사용자가 AI 분석이 진행되고 있음을 이해하고 기다릴 수 있게 하는 전환 화면이다.

### 표시 단계

```text
1. 견적서 파일 확인
2. 금액/단가/부가세 항목 추출
3. Size/Spec 차이 확인
4. 동일 카테고리 여부 확인
5. 비요청 옵션 분리
6. 결측치 처리 정책 적용
7. 공급사별 장단점 생성
8. AI 근거 요약 생성
9. 비교 대시보드 구성
```

### UX 방향

단순 로딩 스피너보다 단계별 진행 상태를 보여준다.

예:

```text
완료: 견적서 파일 확인
완료: 금액 항목 추출
진행 중: 비요청 옵션 분리
대기: AI 근거 요약 생성
```

분석이 완료되면 자동으로 비교 대시보드로 이동한다.

## 4.5 비교 대시보드

### 목적

공급사 견적을 비교하고, AI 추천 근거를 확인하며, 담당자가 최종 공급사를 선정하는 메인 화면이다.

현재 구현된 프로토타입은 이 화면에 해당한다.

### 주요 영역

- 프로젝트 상단 정보
- 공급사 매칭 현황
- 견적 비교 테이블
- AI 근거 요약
- 공급사별 장단점
- 검토 메모
- 최종 선정
- 보고서/내보내기

### 프로젝트 상단 정보

표시:

- 프로젝트명
- 고객사/회사명
- 설치 위치
- 프로젝트 일정
- 비교 마감일
- 현재 단계
- 검토 프리셋

날짜:

- 프로젝트 일정, 비교 마감일은 Date Picker로 설정
- 대시보드에서는 D-day와 진행 상태로 표시

### 견적 비교 테이블

항상 보여야 하는 필수 정보:

- 견적금액 VAT 별도
- 업력
- 매출액 (3년 평균)
- 프로젝트 수 (3년 평균)
- 회사 위치

접고 펼칠 수 있는 하위 항목:

- 디스플레이 H/W
- 시스템 장비
- 설치비

추가로 표시할 수 있는 정보:

- 카테고리
- 제품명
- Size
- Spec
- 동일 Spec 여부
- 특이사항

### 비교표 수정 기능

담당자가 비교표를 수정할 수 있어야 한다.

가능 기능:

- 셀 값 수정
- 행 추가
- 행 삭제
- 항목명 수정
- 비교 제외 표시
- 확인 필요 표시
- 비요청 옵션 표시
- 수정 사유 입력

AI 재분석 시 사용자가 수정한 비교표 데이터를 기반으로 다시 추천한다.

### AI 재분석

AI 재분석은 원본 견적서만 다시 분석하는 것이 아니라 현재 비교표 상태를 함께 반영한다.

입력 데이터:

- 원본 견적서 추출값
- 사용자 수정값
- 추가한 항목
- 비교 제외 항목
- 검토 메모
- 비교 정책

재분석 후 갱신되는 영역:

- AI 근거 요약
- 공급사별 장단점
- 최종 추천 공급사
- 확인 필요 항목
- 공급사 카드 태그/상태

주의:

- AI가 사용자가 수정한 셀 값을 임의로 덮어쓰면 안 된다.
- AI는 수정된 기준을 바탕으로 추천과 설명을 다시 생성한다.

## 4.6 보고서 미리보기/내보내기

### 목적

고객 또는 내부 결재 라인에 공유할 수 있는 형태로 검토 결과를 정리한다.

### 지금 단계에서 고려해야 하는 이유

보고서 생성은 나중 기능처럼 보이지만, 대시보드에서 어떤 정보를 구조화해야 하는지에 영향을 준다.

예:

- 최종 추천 이유
- 비교 기준
- 제외한 옵션
- 확인 필요 항목
- 원본 견적 기준
- 담당자 메모

따라서 MVP에서 완성된 보고서 기능을 만들 필요는 없지만, 보고서에 들어갈 데이터 구조는 지금부터 고려해야 한다.

### MVP 처리 방식

초기에는 아래 정도만 구현한다.

- 고객 보고서 미리보기 버튼
- 보고서에 들어갈 주요 섹션 미리보기
- PDF/Excel 내보내기는 후순위

### 보고서 구성 초안

```text
1. 프로젝트 개요
2. 고객 요청사항
3. 비교 대상 공급사
4. 견적 비교 요약
5. AI 추천 근거
6. 공급사별 장단점
7. 확인 필요 항목
8. 최종 선정 결과
9. 유의사항
```

## 5. 반응형 전략

### 5.1 PC

가장 중요한 기준 화면이다.

- 프로젝트 목록: 카드/테이블 혼합
- 생성 Wizard: 좌측 단계 + 우측 입력 폼
- 대시보드: 메인 비교표 + 우측 AI 패널

### 5.2 태블릿

정보 구조는 유지하되, 좌우 분할을 줄인다.

- 프로젝트 목록: 2열 카드
- 생성 Wizard: 상단 단계 + 입력 폼
- 대시보드: 우측 패널을 아래로 이동

### 5.3 모바일

전체 기능보다는 조회 중심으로 대응한다.

- 프로젝트 목록: 1열 카드
- 생성 Wizard: 한 단계씩 세로 진행
- 대시보드: 요약 카드 중심
- 비교표: 가로 스크롤 또는 카드형 전환

## 6. 히스토리 구조

히스토리는 세 가지로 나눈다.

### 6.1 프로젝트 히스토리

로그인 후 프로젝트 목록에서 보는 이전 검토 건이다.

예:

- PV-2025-0421 삼성전자 반도체 사이니지 구축
- PV-2025-0422 회의실 LED 교체

### 6.2 프로젝트 내부 활동 기록

특정 프로젝트 안에서 누가 무엇을 했는지 기록한다.

예:

- 견적서 업로드
- AI 분석 완료
- 비교표 값 수정

### 6.3 AI 재분석 버전 기록

AI 재분석이 여러 번 실행될 경우 결과 버전을 남긴다.

초기 MVP에서는 자동 기록만 남긴다.

예:

- 최초 분석
- 설치비 확인 후 재분석
- 비요청 옵션 제외 후 재분석

나중에 필요하면 재분석 전 사용자가 버전 이름을 저장하도록 한다.

## 7. 아직 결정이 필요한 것

- AI 재분석 결과를 버전 이름까지 저장할지
- 비교표 수정 기능의 범위를 어디까지 둘지
